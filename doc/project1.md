## [프로젝트 I] 프로젝트 정의서

**(1) 프로젝트 명칭 정하기**
* **코드버디**: 실시간 공동 편집 및 AI 코드 리뷰 지원 플랫폼 구축 프로젝트

**(2) 개발 시스템 명칭 정의하기**
* **지능형 실시간 협업 코드 에디터**

**(3) 개발 시스템에 대하여 간략히 설명하기**
* 본 시스템은 웹 브라우저 환경에서 여러 개발자가 동시에 접속하여 소스 코드를 편집하고 피드백을 실시간으로 주고받을 수 있는 협업 전용 플랫폼입니다. 개발 중 발생할 수 있는 에러를 방지하기 위해 AI가 코드를 실시간으로 분석하여 잠재적 오류를 탐지하며, 형상 관리 기능을 통합하여 프로젝트의 모든 변경 이력을 체계적으로 관리합니다. 이를 통해 개발 팀의 의사소통 비용을 줄이고 소프트웨어의 전체적인 완성도를 높이는 것을 목적으로 합니다.

**(4) 개발 시스템에서 제공하는 핵심 기능 5개 이상 나열하기**
1. **실시간 동시 편집**: 웹 소켓 기술을 활용하여 여러 사용자가 하나의 파일을 지연 없이 동시에 수정하고 서로의 작업 위치를 확인하는 기능.
2. **AI 코드 어드바이저**: 코드 작성 중 실시간으로 문법 오류를 검출하고, 성능 및 유지보수성 향상을 위한 최적화 로직을 제안하는 기능.
3. **인라인 코드 리뷰 및 스레드 채팅**: 특정 코드 라인에 직접 댓글을 달아 팀원과 소통하며, 토론 내용을 기록으로 보존하는 기능.
4. **Git/GitHub 통합 연동**: 별도의 터미널 작업 없이 작업 결과물을 원격 저장소로 즉시 전송하고 Pull Request를 생성 및 관리하는 기능.
5. **실시간 정적 분석 엔진**: 변수 명명 규칙 준수 여부 및 데이터 단위 불일치와 같은 논리적 결함을 사전에 탐지하여 알림을 제공하는 기능.
6. **협업용 코드 스니펫 저장소**: 팀 내에서 공통으로 사용되는 코드 조각이나 템플릿을 저장하고 공유하여 개발 생산성을 높이는 기능.

**(5) 기존 유사 소프트웨어에 대하여 조사하고 분석하기**
* **Visual Studio Code (Live Share)**: 로컬 설치형 도구로서 강력한 플러그인 생태계를 보유하고 있으나, 모든 사용자가 동일한 프로그램을 설치하고 확장 기능을 맞춰야 하는 환경 설정의 번거로움이 있음.
* **Replit**: 클라우드 기반 IDE로 뛰어난 접근성과 협업 기능을 제공하지만, 대규모 프로젝트를 위한 전문적인 코드 리뷰 도구 및 정밀한 품질 검사 기능과의 연동이 다소 부족함.
* **GitHub**: 강력한 저장소 연동과 독립적인 개발 환경을 제공하지만, 실시간 동시 수정보다는 비동기식 리뷰(Pull Request) 중심의 협업 모델에 특화되어 있음.
* **본 프로젝트의 차별점**: 기존 서비스들의 강점인 '웹 기반의 높은 접근성'에 'AI를 통한 실시간 품질 교정' 기능을 결합하였습니다. 코드 통합 전 단계에서부터 오류를 차단하고 실시간으로 지식을 공유할 수 있는 환경을 구축하여 협업의 질을 한 단계 높이고자 합니다.
