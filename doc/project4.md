# [실습 프로젝트 VI] 요구사항 정의서

## 1. 개요
*   **1.1 문서의 목적**: 본 문서는 실시간 AI 협업 에디터 CodeBuddy의 사용자 및 개발 관련 요구사항을 명확히 정의하여 프로젝트의 범위와 목표를 확립하는 데 목적이 있습니다
*   **1.2 프로젝트 개요**: 웹 브라우저 기반에서 여러 개발자가 동시에 코드를 편집하고, AI를 통해 실시간 코드 리뷰 및 품질 검사를 수행하는 지능형 협업 플랫폼을 개발합니다.
*   **1.3 관련 문서, 용어 및 약어**: 프로젝트 정의서(Project 1), 프로젝트 관리 계획서(Project 3), MSA(마이크로서비스 아키텍처), WebSocket
## 2. 요구사항
각 요구사항은 식별자(ID)를 부여하고 단문 형태로 명확하게 기술합니다.

*   **2.1 기능적 요구사항**
    *   **F-01**: 시스템은 웹소켓을 통해 여러 사용자의 편집 내용을 실시간으로 동기화해야 합니다
    *   **F-02**: 시스템은 작성 중인 코드의 문법 오류를 탐지하여 사용자에게 즉시 알림을 제공해야 합니다
    *   **F-03**: 시스템은 특정 코드 라인에 대해 인라인 리뷰 및 스레드 채팅 기능을 제공해야 합니다
    *   **F-04**: 시스템은 별도의 터미널 없이 GitHub 저장소로 코드를 푸시하거나 PR을 생성하는 기능을 제공해야 합니다

*   **2.2 비기능적 요구사항**
    *   **NF-01**: 시스템은 서비스의 독립적 배포와 확장을 위해 MSA 구조로 설계되어야 합니다
    *   **NF-02**: 시스템은 개인정보 및 소스 코드 보호를 위해 AWS IAM 기반의 권한 관리 기능을 제공해야 합니다
*   **2.3 인터페이스 요구사항**
    *   **I-01**: 시스템은 사용자가 별도 설치 없이 브라우저를 통해 즉시 접속 가능한 웹 인터페이스를 제공해야 합니다
    *   **I-02**: 시스템은 Kakao Social Login 연동을 통한 사용자 인증 인터페이스를 포함해야 합니다

## 3. 품질 요구사항
프로젝트에서 식별된 주요 품질 요소를 기준으로 정의합니다

*   **3.1 응답성**: 실시간 편집 시 데이터 전송 지연 시간은 **100ms 미만**을 유지해야 합니다
*   **3.2 정확성**: AI 코드 어드바이저의 오류 탐지 및 수정 제안 정확도는 **90% 이상**이어야 합니다
*   **3.3 안정성**: 특정 마이크로서비스 장애 시에도 핵심 에디터 기능은 중단 없이 운영되어야 합니다

## 4. 개발 요구사항
*   **4.1 개발 지원 소프트웨어**: Spring Boot(Backend), React(Frontend), PyTorch(AI 모델), AWS 클라우드 서비스를 활용합니다
*   **4.2 하드웨어 요구사항**: AI 모델 추론을 위한 GPU 가속 인스턴스(AWS) 및 고성능 개발 워크스테이션(Intel i5-13600KF, RTX 4070급)이 필요합니다

## 5. 기타 고려 사항
*   기존 유사 소프트웨어와의 차별화를 위해 AI 기반의 실시간 품질 교정 기능을 고도화합니다
*   서버 부하 급증에 대비하여 오토스케일링 대응 로직을 마련합니다
