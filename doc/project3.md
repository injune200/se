# 프로젝트 관리 계획서: CodeBuddy

## 1. 개요 (Introduction)
*   **1.1 문서의 목적**: 본 문서는 실시간 AI 협업 에디터인 CodeBuddy 개발 프로젝트의 전반적인 수행 계획을 정의하며, 팀원 간의 원활한 소통과 품질 보증을 위한 가이드라인으로 활용됩니다.
*   **1.2 프로젝트 개요**: 웹 브라우저 기반의 실시간 공동 편집 환경을 구축하고, AI 코드 리뷰 및 형상 관리 통합을 통해 개발 생산성을 극대화하는 시스템을 개발합니다.
*   **1.3 관련 문서, 용어 및 약어**: 프로젝트 정의서(Project I), 요구사항 명세서, MSA(Microservices Architecture), WebSocket.

## 2. 개발 계획 (Development Plan)
*   **2.1 프로세스 모델 및 개발 방법론**: 신속한 피드백 반영을 위해 **애자일(Agile) 모델**을 채택하며, 스프린트 단위로 기능을 확장합니다.
*   **2.2 작업 분할도 (WBS)**: 에디터 코어 개발 → WebSocket 서버 구축 → AI 모델 연동 → Git 연동 모듈 개발 → 통합 테스트 순으로 진행합니다.
*   **2.3 소요 자원 (Staffing, Cost)**: 기간과 비용에 제한을 두지 않고 필요에 따라 유연하게 투입합니다. 클라우드 인프라 운영 비용과 AI API 사용료를 포함합니다.
*   **2.4 일정 계획 (Gantt Chart)**: MVP(최소 기능 제품) 개발 완료 후 상시 고도화 단계로 진입합니다.

## 3. 조직 구성 (Organization)
*   **3.1 팀 구조**: 전문성 중심의 **분산형 팀 구조**를 선택하여 각 모듈(FE, BE, AI) 담당자가 독립적으로 의사결정을 수행하되 유기적으로 협업합니다.
*   **3.2 책임 및 역할**:
    *   **FE팀**: 에디터 UI 및 실시간 동기화 인터페이스 담당.
    *   **BE팀**: 서버 아키텍처 및 데이터베이스, 보안 관리 담당
    *   **AI팀**: 실시간 코드 분석 및 리팩토링 로직 개발 담당.

## 4. 기술 관리 계획 (Technical Management)
*   **4.1 버전 관리**: 소스 코드는 파일명 앞에 버전을 기재하여 **GitHub**를 통해 관리하며, 문서는 파일명 뒤에 날짜와 제/개정 이력을 남깁니다.
*   **4.2 형상 관리**: 산출물 Naming 규칙인 `[CodeBuddy]문서명_YYMMDD_Doc-버전` 형식을 철저히 준수합니다.
*   **4.3 기술 관리**: 기능 변경 시 담당 파트원의 합의가 필수적이며, 모든 변경 사항은 기록물로 보존하여 기술 부채를 방지합니다.

## 5. 품질 관리 계획 (Quality Control)
*   **5.1 품질 목표**: 실시간 동기화 지연 시간 100ms 미만 유지 및 AI 코드 리뷰의 정확도 90% 이상 확보.
*   **5.2 검토 방법 및 주기**: 매 스프린트 종료 후 정기적인 코드 리뷰 및 피어 인스펙션(Peer Inspection)을 수행합니다.
*   **5.3 품질 관리 적용 기법**: AI를 활용한 실시간 정적 분석과 단위 테스트 자동화를 병행하여 결함을 사전에 탐지합니다.
*   **5.4 리스크 관리**: 서버 부하 급증 시 오토스케일링(Auto-scaling) 대응 및 AI 모델의 오답 가능성에 대한 예외 처리 로직을 마련합니다.

## 6. 개발 환경 (Development Environment)
*   **6.1 요구되는 소프트웨어 스펙**: VS Code, Node.js, Spring Boot, PyTorch, AWS SDK.
*   **6.2 요구되는 하드웨어 스펙**: 고성능 개발 워크스테이션 및 GPU 연산용 AWS 인스턴스.
*   **6.3 개발 공간 및 보안 요구사항**: 클라우드 기반 협업 공간 활용 및 AWS IAM을 통한 철저한 접근 권한 관리.

## 7. 개발 산출물 (Deliverables)
*   **7.1 산출물 목록**: 요구사항 명세서, 설계 도면, 소스 코드 저장소, API 가이드 문서, 사용자 매뉴얼.

## 8. 기타 고려 사항
*   인터넷 유사 시스템(Replit, VS Code Live Share)의 장점을 벤치마킹하여 웹 기반 접근성과 고도의 코드 품질 관리 기능을 동시 구현합니다.
