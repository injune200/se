# [과제5] 테스트 결과 보고서

## 1. 서론

### 1.1 문서 목적 및 범위
본 문서의 목적은 지능형 실시간 협업 코드 에디터 플랫폼 **'CodeBuddy'** 프로젝트에 대한 테스팅을 수행하고 그 결과를 기록하는 데 있다. 본 문서에서는 시스템의 요구사항을 기반으로 설계된 테스트 기준, 테스트 범위, 구체적인 테스트 케이스 및 최종 테스트 결과를 명시한다.

### 1.2 프로젝트 개요
본 시스템은 웹 브라우저 환경에서 여러 개발자가 동시에 접속하여 소스 코드를 편집하고 피드백을 실시간으로 주고받을 수 있는 협업 전용 플랫폼이다. 개발 중 발생할 수 있는 에러를 방지하기 위해 AI가 코드를 실시간으로 분석하여 잠재적 오류를 탐지하며, 형상 관리 기능을 통합하여 프로젝트의 모든 변경 이력을 체계적으로 관리한다. 이를 통해 개발 팀의 의사소통 비용을 줄이고 소프트웨어의 전체적인 완성도를 높이는 것을 목적으로 한다.

### 1.3 용어 정의
| 용어 | 설명 |
| :--- | :--- |
| **서버사이드 (Server-side)** | 서버 측 시스템을 뜻하며, 본 프로젝트에서는 마이크로서비스 아키텍처(MSA) 기반의 편집 엔진, AI 분석 서비스, 인증 및 협업 모듈과 데이터베이스 연산 등을 수행한다. |
| **클라이언트 (Client)** | 사용자 눈에 보이는 시스템을 뜻하며, 본 프로젝트에서는 별도의 설치 없이 브라우저를 통해 접속 가능한 웹 인터페이스 시스템을 의미한다. |
| **WebSocket** | 웹 브라우저와 서버 간의 전이중 양방향 통신을 지원하는 프로토콜로, 다중 사용자가 하나의 파일을 지연 없이 동시에 수정하고 동기화하기 위해 사용된다. |
| **코드 스멜 (Code Smell)** | 작동은 하지만 가독성이 떨어지거나, 중복 구조 등으로 인해 향후 유지보수에 문제를 일으킬 가능성이 높은 코드 패턴을 의미한다. |

2. [프로젝트 III] 프로젝트 관리 계획서 (Project 3)

## 2. 테스트 개요

### 2.1 테스트 범위
본 테스트는 시스템 내부 소스 코드의 구조를 모른다는 가정하에, 정의된 요구사항 명세서를 기반으로 기능의 정상 동작 여부를 확인하는 명세 기반 **블랙박스 테스트(Black-box Test)**로 진행된다. 테스트 범위는 다음과 같다.

* **Web 기반 에디터 및 협업 시스템**: 웹 브라우저 접속 인터페이스 동작, 웹소켓 세션 관리, 동시 편집 및 인라인 리뷰 기능의 정확성 검증
* **AI 어드바이저 엔진**: 비동기 코드 분석, 문법 오류 및 코드 스멜 탐지 성능, 가독성 향상을 위한 리팩토링 제안 출력 검증
* **형상 관리 및 권한 인프라**: 카카오 소셜 로그인 연동, 데이터베이스(MySQL) 입출력, 터미널 없는 Git/GitHub 저장소 연동 기능 검증

### 2.2 테스트 항목 및 통과 기준

#### 2.2.1 기능별 테스트 항목
| 기능별 테스트 항목 | 통과 기준 |
| :--- | :--- |
| **카카오 로그인 기능** | Kakao Social Login 연동을 통해 올바른 인증 정보 토큰이 유효할 때만 워크스페이스로 정상 진입한다. |
| **실시간 동시 편집 기능** | 다중 사용자가 동일 파일 편집 세션에 참여 시, 웹소켓을 통해 편집 내역이 커서 위치와 함께 실시간 동기화된다. |
| **AI 코드 어드바이저 기능** | 작성 중인 코드의 실시간 문법 오류(Syntax Error)를 탐지하고, 중복 코드나 미사용 변수 분석 후 수정 로직을 UI에 출력한다. |
| **인라인 리뷰 및 채팅 기능** | 에디터 내 특정 코드 라인에 댓글(리뷰)을 작성할 수 있으며, 팀원 간의 대댓글(스레드 채팅)이 실시간으로 갱신된다. |
| **Git/GitHub 연동 기능** | 별도의 로컬 터미널 명령어를 사용하지 않고, UI 상의 버튼 클릭만으로 원격 저장소 푸시(Push) 및 PR 생성이 완료된다. |

#### 2.2.2 비기능별 테스트 항목
| 비기능별 테스트 항목 | 통과 기준 |
| :--- | :--- |
| **응답성 (Performance)** | 다중 편집 상황에서 클라이언트 간 데이터 전송 및 화면 동기화 지연 시간이 **100ms 미만**을 유지해야 한다. |
| **정확성 (Accuracy)** | AI 코드 어드바이저가 코드의 잠재적 문법 오류 및 최적화 요소를 올바르게 잡아내는 탐지 정확도가 **90% 이상**이어야 한다. |
| **보안 및 권한 관리 (Security)** | AWS IAM 및 세션 권한 제어를 통해, 로그인하지 않거나 초대받지 않은 외부 사용자는 프로젝트 소스에 접근할 수 없어야 한다. |
| **안정성 및 장애 격리 (Reliability)** | 마이크로서비스 아키텍처(MSA) 설계에 따라, 특정 모듈(예: AI 분석 엔진)에 장애가 발생하더라도 핵심 기능인 실시간 에디터 및 동기화는 중단 없이 운영되어야 한다. |

### 3.2 테스트 케이스 유형
* **동작 및 인터페이스 검증**: 카카오 로그인 수행 및 에디터 컴포넌트 렌더링, 인라인 리뷰 UI 검증

* **AI 로직 및 데이터 예외처리 검증**: 비정상 코드 입력 시 AI 모델의 피드백 도출 및 특정 마이크로서비스 가동 중단 시 장애 격리 여부 검증

## 4. 테스트 예외사항

### 4.1 중단 기준과 재개 조건
* **테스트 중단 기준**: 대규모 동시 타이핑 테스트 중 웹소켓 세션 풀이 만료되거나, 트래픽 폭증으로 인해 동기화 지연 시간이 지속해서 **500ms를 초과**하여 정상적인 협업 코딩 시나리오 진행이 불가능할 경우 테스트를 즉시 중단한다.
* **테스트 재개 조건**: AWS 인프라의 가상 네트워크 및 로드밸런서(ALB) 설정을 점검하고, 에디터 세션 관리 서버의 스케일 아웃(Scale-out) 배포가 완료되어 데이터 전송 지연 시간이 통과 기준인 **100ms 미만**으로 회복되었음이 모니터링 도구로 확인될 때 테스트를 재개한다.

### 7.1 AI 정밀도(T7) 미달 요인 분석 및 조치 계획
* **요인 분석**: 비기능 요구사항인 오류 탐지 및 최적화 제안 정밀도 테스트(T7)에서 목표치인 90%에 다소 미달하는 **88.5%**의 결과가 도출되었다. 세부 로그 분석 결과, 복잡하게 중첩된 제어문(Nested Loop) 및 다중 스레드 동기화 관련 코드 패턴 환경에서 잠재적 데들락(Deadlock) 위험성을 탐지하는 규칙의 오탐율이 높은 것으로 파악되었다.
